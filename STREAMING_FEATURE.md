# 🌊 Streaming Chat Feature

## Overview

The chat interface now supports real-time streaming responses! Instead of waiting for the complete answer, you'll see the AI's response being generated token-by-token, just like ChatGPT.

## What's New

### Backend Changes

**New Endpoint:** `POST /chat/stream`
- Returns Server-Sent Events (SSE) for real-time streaming
- Streams tokens as they're generated by the LLM
- Provides status updates during processing

**Stream Event Types:**
```json
// Status update
{"type": "status", "message": "Searching document..."}

// Context information
{"type": "context", "chunks": 5, "sources": ["document.pdf"]}

// Token (word/phrase being generated)
{"type": "token", "content": "The answer is "}

// Completion
{"type": "done", "conversation_id": "...", "sources": [...]}

// Error
{"type": "error", "message": "..."}
```

### Frontend Changes

**Updated ChatPage:**
- Real-time token display as they arrive
- Animated blinking cursor during streaming
- Smooth scrolling to follow the response
- Better loading states

**New API Function:** `sendMessageStream()`
- Handles SSE connection
- Parses streaming chunks
- Calls callback for each token

## How It Works

### Backend Flow

```
User sends question
    ↓
Backend retrieves relevant chunks from FAISS
    ↓
Sends status: "Searching document..."
    ↓
Sends context info: chunks count, sources
    ↓
Sends status: "Generating answer..."
    ↓
Streams LLM response token by token
    ↓
Sends completion with conversation_id
```

### Frontend Flow

```
User clicks send
    ↓
Shows "Thinking..." status
    ↓
Connects to /chat/stream endpoint
    ↓
Receives tokens and appends to message
    ↓
Shows blinking cursor during streaming
    ↓
Completes and adds to message history
```

## Technical Details

### Server-Sent Events (SSE)

The backend uses SSE format:
```
data: {"type": "token", "content": "Hello"}

data: {"type": "token", "content": " world"}

data: {"type": "done", "conversation_id": "..."}

```

Each event is prefixed with `data: ` and separated by double newlines.

### LangChain Streaming

Uses LangChain's `astream()` method:
```python
async for chunk in chain.astream(inputs):
    if chunk.content:
        yield {"type": "token", "content": chunk.content}
```

### Frontend Parsing

Uses Fetch API with ReadableStream:
```javascript
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const text = decoder.decode(value);
  // Parse SSE format and extract JSON
}
```

## Troubleshooting

### Streaming Not Working

**Check backend logs:**
```
INFO: Creating Groq LLM with model: llama-3.3-70b-versatile
```

**Check browser console:**
- Should see SSE chunks being received
- No CORS errors
- No network errors

**Common Issues:**

1. **Buffering by proxy/nginx:**
   - Add `X-Accel-Buffering: no` header (already included)

2. **CORS errors:**
   - Ensure frontend URL is in `CORS_ORIGINS`

3. **Connection timeout:**
   - Check firewall settings
   - Ensure backend is running

### Fallback to Non-Streaming

If streaming fails, the app will show an error. You can modify the code to fallback to the non-streaming endpoint:

```javascript
try {
  await sendMessageStream(...);
} catch (err) {
  // Fallback to non-streaming
  const result = await sendMessage(...);
}
```
