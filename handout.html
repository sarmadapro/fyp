Overview & Pipeline

Multi-tenant SaaS platform for voice-based document interaction via RAG. Five-stage pipeline: voice input → STT (Faster-Whisper) → retrieval (FAISS + MiniLM) → LLM (Groq/Ollama) → TTS (Kokoro). Three microservices: FastAPI (8000), STT (8001), TTS (8002). Per-client isolation with SQLAlchemy ORM and FAISS indices at data/clients/{client_id}/.

System Architecture Diagram

Core Capabilities

  • Multi-tenant isolation — Per-client FAISS index, SHA-256 API keys
  • Real-time streaming — SSE/WebSocket for voice responses
  • Embeddable widget — Third-party integration via single script
  • Analytics & memory — Per-stage latency, conversation history
  • LLM fallback — Groq → Ollama if API unavailable
